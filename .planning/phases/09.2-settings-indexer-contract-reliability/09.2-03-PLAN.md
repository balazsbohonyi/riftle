    **Fix 2: Notify timer thread in `set_settings_cmd`**

    Replace the current `set_settings_cmd` function with:

    ```rust
    #[tauri::command]
    pub fn set_settings_cmd(
        app: tauri::AppHandle,
        data_dir: tauri::State<std::path::PathBuf>,
        settings: Settings,
    ) -> Result<(), String> {
        set_settings(&app, &data_dir, &settings);
        // Notify the background timer thread of the new interval.
        // Uses try_state (not State parameter) so this command is safe in all build targets,
        // even if the timer was not started (e.g. mobile builds or tests).
        // Pattern matches existing app.try_state usage in search.rs.
        use std::sync::{Arc, Mutex};
        if let Some(timer_state) = app.try_state::<Arc<Mutex<std::sync::mpsc::Sender<crate::indexer::TimerMsg>>>>() {
            if let Ok(tx) = timer_state.lock() {
                let _ = tx.send(crate::indexer::TimerMsg::SetInterval(settings.reindex_interval));
            }
        }
        Ok(())
    }
    ```

    IMPORTANT: The `use std::sync::{Arc, Mutex};` is already imported at the top of `lib.rs` but NOT in `store.rs`. In `store.rs`, `Arc` and `Mutex` are not currently imported. Either add them as a local `use` inside the function body (as shown above) or add them to the top-level imports. The local `use` inside the function is acceptable — it avoids widening the import footprint of the file.

    PITFALL: Do NOT add `timer_tx: tauri::State<...>` as a function parameter. Use `app.try_state()` instead. Adding a `State<T>` parameter when `T` is only managed in `#[cfg(desktop)]` would cause a runtime panic in non-desktop builds. The `try_state` approach is safe — it returns `None` if the state was never managed.

    PITFALL: The full type path for `try_state` must exactly match what was managed in `lib.rs`. In Plan 02, `app.manage(Arc::new(Mutex::new(timer_tx)))` where `timer_tx: mpsc::Sender<TimerMsg>` — so the managed type is `Arc<Mutex<mpsc::Sender<crate::indexer::TimerMsg>>>`. Match this exactly in `try_state`.
