## Tests

### 1. System commands work after saving settings
expected: Open Settings (Ctrl+,). Make any change (e.g. toggle Show Path on and off). Save. Close Settings. In the launcher, type "> " — the system commands (Lock Screen, Shutdown, Restart, Sleep) should still appear. Previously, saving settings would silently wipe the system_tool_allowlist from the stored JSON, which could cause system commands to stop appearing after a save.
result: pass

### 2. Manual re-index respects additional paths
expected: In Settings > Search, add a custom path that contains .exe or .lnk files (or has a known app). Click "Re-index now". After indexing completes, search for an app that is only in that custom path — it should appear in results. This confirms reindex() calls get_settings() for live paths instead of Settings::default() (which returns empty additional_paths).
result: pass

### 3. Background re-indexing disabled at interval 0
expected: If your settings have a reindex interval of 0 (disabled), the app should not trigger background re-indexing automatically. With interval 0: launch the app, leave it running for a couple minutes — it should not show any indexing activity and should not momentarily stutter or lag (which would indicate an accidental zero-duration loop firing constantly). Note: this is hard to observe directly — skip if you don't have a way to detect background indexing.
result: pass

### 4. App launches correctly after settings save
expected: After opening Settings and saving (no changes needed), close Settings and verify the launcher still works: pressing Alt+Space shows the launcher, searching for a known app returns results, and pressing Enter launches it successfully. This is a basic sanity check that the store.rs changes didn't break settings persistence.
result: pass
