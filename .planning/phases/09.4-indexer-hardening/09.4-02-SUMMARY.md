## Accomplishments
- Added `normalize_for_exclusion()` private helper that canonicalizes paths, falls back to raw PathBuf on error, lowercases, and strips trailing separators

- Updated `resolve_lnk` to use a 32,767 u16 buffer (`EXTENDED_MAX_PATH`) instead of MAX_PATH (260) — future-proofs against extended-length path targets

## Decisions Made
- normalize_for_exclusion uses `canonicalize()` with `unwrap_or_else(|_| p.to_path_buf())` fallback so excluded dirs that no longer exist on disk still work (they fall back to raw+lowercase comparison)
- Trailing separator strip via `trim_end_matches(['/', '\\'])` handles both separator styles
- WalkDir `follow_links(false)` only affects symlinks encountered during traversal; `follow_root_links` still defaults to true so the root itself (if a symlink) is followed — this matches the intended behavior
- EXTENDED_MAX_PATH declared as `const` inline in `resolve_lnk` body rather than at module scope — keeps the rationale comment co-located with the allocation
