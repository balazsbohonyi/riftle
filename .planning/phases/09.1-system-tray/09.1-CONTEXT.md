# Phase 09.1: System Tray + OS Menu - Context

**Gathered:** 2026-03-10
**Status:** Ready for planning

<domain>
## Phase Boundary

Show the app in the Windows system tray using the app's default icon. Right-clicking the tray icon must show a native OS context menu with the same options as the launcher context menu.

</domain>

<decisions>
## Implementation Decisions

### Tray click behavior
- Single left-click toggles the launcher: show if hidden, hide if visible.
- Double-click always shows/focuses the launcher (no toggle on double-click).
- Tray-triggered show uses the same hotkey-style flow: clear query/results, replay animation, center window, focus input.
- If a tray action resolves to "show" and the launcher is already visible, focus and center it (no hide).

### Tray menu contents
- Tray menu contains exactly two items: "Settings" and "Quit Launcher".
- Order is "Settings" first, then "Quit Launcher".
- Labels match the launcher context menu text exactly.
- Actions match launcher menu behavior: Settings opens/focuses the settings window, Quit exits the app.

### Tray tooltip
- Tooltip text: "Riftle Launcher".
- Tooltip is static (no dynamic status).
- Tooltip does not include the hotkey.
- Tray icon appears in dev builds as well as release.

### Claude's Discretion
- Exact Tauri tray API usage and event wiring (tray icon creation, click event mapping).
- Where to store any tray-related state if needed for click handling.

</decisions>

<specifics>
## Specific Ideas

- Tray menu must be a native OS context menu (not a custom HTML overlay).
- Use the app's default/bundled icon for the tray icon.

</specifics>

<code_context>
## Existing Code Insights

### Reusable Assets
- `src/App.vue`: launcher context menu already has "Settings" and "Quit Launcher" items and invokes `open_settings_window` and `quit_app`.
- `src-tauri/src/lib.rs`: `open_settings_window` and `quit_app` Tauri commands are already registered.
- `src-tauri/tauri.conf.json`: bundle icons are declared; use the default app icon for the tray.

### Established Patterns
- Launcher show flow uses the `"launcher-show"` event to clear query, replay animation, center, and focus.
- Window show/hide and focus are handled via `getCurrentWindow()` in App.vue and Tauri window APIs in Rust.

### Integration Points
- `src-tauri/src/lib.rs` setup is the natural place to create and register the tray icon and menu.
- Tray menu items should call the same commands as the launcher context menu: `open_settings_window` and `quit_app`.

</code_context>

<deferred>
## Deferred Ideas

- Additional tray menu items (e.g., "Show Launcher") are out of scope for this phase.

</deferred>
