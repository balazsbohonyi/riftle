# Phase 09.9: Show Launcher on Default Monitor - Context

**Gathered:** 2026-06-14
**Status:** Ready for planning

<domain>
## Phase Boundary

Fix multi-monitor launcher positioning: the launcher currently appears on whichever monitor the hidden window last occupied (`current_monitor()`), but should appear on the Windows **primary/default monitor**. Add an optional "Show where cursor is" toggle in Settings to override this. Apply the same primary-monitor fix to the Settings window, and update CHANGELOG.md with the unreleased feature entry.

</domain>

<decisions>
## Implementation Decisions

### Default monitor behavior
- **Launcher:** Always show on the Windows primary/default monitor, centered. Replace `current_monitor()` with `primary_monitor()` in `show_positioned_launcher()` (`lib.rs:113-116`).
- **Settings window:** Always show on the primary/default monitor, always centered. Replace the `SettingsCentered` one-time-center guard with always-center behavior.

### "Show where cursor is" toggle
- **Placement:** Appearance section in Settings, as a `Row` + `Toggle` using existing UI primitives.
- **Label:** "Show where cursor is"
- **Default:** OFF (unchecked)
- **Behavior:** When ON, the launcher appears on the monitor containing the mouse cursor instead of the primary monitor. Query cursor position at summon time only (no continuous tracking). On re-show (window hidden then shown again), reposition to the cursor's current monitor.
- **Scope:** Launcher only â€” the Settings window does NOT follow this toggle; it stays on the primary monitor.

### CHANGELOG
- **Location:** Under the `## Unreleased â€” In development` section (do not modify the existing `## 1.0.0-beta.1` entry).
- **Entry:** "Launcher and Settings now appear on the primary monitor by default. Optionally show the launcher on the monitor where the mouse cursor is."
