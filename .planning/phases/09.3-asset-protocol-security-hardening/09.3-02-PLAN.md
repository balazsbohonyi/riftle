    ```rust
    pub fn validate_icon_filename(filename: &str) -> bool {
        // Allow the two static icons
        if filename == "generic.png" || filename == "system_command.png" {
            return true;
        }
        // Require exactly 16 lowercase hex chars + ".png" = 20 chars total
        if filename.len() != 20 {
            return false;
        }
        let (hex_part, ext) = filename.split_at(16);
        ext == ".png" && hex_part.chars().all(|c| matches!(c, '0'..='9' | 'a'..='f'))
    }
    ```

    The full updated `security` block becomes:

    ```json
    "security": {
      "csp": "default-src 'self' ipc: http://ipc.localhost; img-src 'self' asset: http://asset.localhost data: blob:",
      "assetProtocol": {
        "enable": true,
        "scope": [
          "$DATA/riftle-launcher/icons/**",
          "$EXE/../data/icons/**"
        ]
      }
    }
    ```

    Scope variable meanings:
    - `$DATA` = `C:\Users\<user>\AppData\Roaming` (bare Roaming folder, NOT bundle-id subdirectory)
    - `$DATA/riftle-launcher/icons/**` covers installed mode where paths.rs hardcodes `riftle-launcher` as the subdirectory name
    - `$EXE` = path to the application executable file
    - `$EXE/../data/icons/**` steps up from the exe file to its parent directory, then into `data/icons/` — covers portable mode where data/ is adjacent to the exe

    IMPORTANT: Do NOT use `$APPDATA` — in Tauri FsScope, `$APPDATA` resolves to `AppData\Roaming\com.riftle.launcher` (bundle-id-based), which does NOT match the hardcoded `riftle-launcher` directory used by paths.rs.

    Do not change any other field in tauri.conf.json.
