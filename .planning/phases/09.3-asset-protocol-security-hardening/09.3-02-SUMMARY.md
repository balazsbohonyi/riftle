# Phase 09.3 Plan 02: enforce validator in search and harden asset protocol scope

**Wired `validate_icon_filename()` into `score_and_rank()` and replaced the global asset wildcard with the two app-owned icon directory scopes required for installed and portable mode.**

## Accomplishments
- Updated `score_and_rank()` so any invalid `icon_path` value is replaced with `generic.png` before the frontend builds an asset URL
- Hardened `src-tauri/tauri.conf.json` from `"scope": ["**"]` to the two explicit icon directory globs
- Confirmed the full Rust suite passes: 51 passed, 0 failed, 2 ignored

## Files Modified
- `src-tauri/src/search.rs` - Applied validator during `SearchResult` construction
- `src-tauri/tauri.conf.json` - Limited `assetProtocol.scope` to installed and portable icon directories

## Decisions Made
- Preserve `system_command.png` as an explicit allowlisted static filename instead of special-casing system command search results elsewhere
- Use `generic.png` as the safe fallback for invalid filenames to avoid broken UI results while still blocking traversal and malformed values
