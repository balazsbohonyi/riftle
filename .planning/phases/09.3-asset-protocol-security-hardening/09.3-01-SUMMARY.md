# Phase 09.3 Plan 01: validator test surface in search.rs

**Added the icon filename validation surface in `search.rs` with eight unit tests covering the full accepted/rejected filename matrix for asset-backed icons.**

## Accomplishments
- Added `validate_icon_filename()` to `src-tauri/src/search.rs` as the central validator for icon filenames used by search results
- Added 8 unit tests for traversal attempts, absolute paths, wrong extensions, uppercase hex, wrong lengths, valid hex names, `generic.png`, and `system_command.png`
- Confirmed `cargo test test_validate_icon` passes with all 8 tests GREEN

## Files Modified
- `src-tauri/src/search.rs` - Added `validate_icon_filename()` and 8 new `test_validate_icon_*` tests

## Decisions Made
- Keep validation logic in `search.rs` instead of `indexer.rs` so corrupted DB values are filtered at read time before reaching `convertFileSrc()`
- Keep the validator narrow: only 16-char lowercase hex PNGs plus the two known static filenames are allowed

## Issues Encountered
- The planned RED scaffold step was collapsed into the implemented validator because the execute-phase subagent did not return usable results; direct execution in the workspace was faster and still preserved the required test coverage

## Verification
- `cargo test test_validate_icon`
