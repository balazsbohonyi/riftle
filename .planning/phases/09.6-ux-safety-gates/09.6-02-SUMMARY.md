# Phase 09.6 Plan 02: Confirmation Overlay for Destructive System Commands Summary

**One-liner:** Keyboard-triggered modal gate that intercepts shutdown/restart in App.vue before `run_system_command` executes, requiring an explicit second confirmation while lock and sleep bypass it entirely.

## What Was Built

**Script additions (Task 1):**
- `CONFIRM_REQUIRED = new Set(['system:shutdown', 'system:restart'])` constant for O(1) intercept lookup
- `confirmBtnRef`, `confirmPending`, `pendingCommand` state alongside context menu state
- `showConfirm(item)` — sets state and auto-focuses confirm button via `nextTick`
- `cancelConfirm()` — clears state and returns focus to search input
- `confirmAction()` — clears state, calls `hideWindow()`, then invokes `run_system_command`
- `launchItem()` modified: intercepts CONFIRM_REQUIRED items and calls `showConfirm` instead of executing
- Escape handler: `confirmPending` check inserted at priority 1.5 (after menuVisible, before hideWindow)
- Enter handler: `confirmPending` pass-through inserted before item selection logic
- `launcher-show` listener: resets `confirmPending` and `pendingCommand` on launcher re-summon

**Template additions (Task 2):**
- Backdrop (`confirm-backdrop`) with `@mousedown.prevent` — blocks focus-loss without closing on outside click
- Card (`confirm-card`) with per-command copy: "Shut down Windows?" / "Restart Windows?"
- Danger button auto-focused via `ref="confirmBtnRef"` — Enter confirms immediately after overlay opens
- Cancel button with `@mousedown.prevent` to prevent focus-loss race

**CSS additions (Task 2):**
- `.confirm-backdrop` at z-index 199 with semi-opaque dark overlay
- `.confirm-card` at z-index 200, centered via `fixed + translate(-50%, -50%)`, 280px wide
- Full button styling: danger variant (accent background), cancel variant (transparent + border)
- `focus-visible` ring on confirm button for keyboard accessibility
