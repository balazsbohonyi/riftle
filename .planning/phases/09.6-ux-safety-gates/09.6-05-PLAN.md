<objective>
Redesign the shutdown/restart confirmation UX (Gap 3). The floating overlay approach was rejected by the user due to three compounding bugs and fundamental design issues. The new design is inline — inside the launcher field itself.

Design specification (from user):
- Remove the `.confirm-backdrop` and `.confirm-card` overlay entirely (template + CSS)
- Replace with a single `.confirm-row` inside `.search-area`
- While `confirmPending=true`: hide the search input (`<input>`) and magnifier icon; show `.confirm-row` instead
- `.confirm-row` layout: a single line with the question text on the left and [Confirm] + [Cancel] buttons on the right
- Question text: "Shut down Windows?" or "Restart Windows?" (same per-command copy, no secondary body text)
- [Confirm] button: danger styling (accent background, white text); auto-focused via `ref="confirmBtnRef"` + nextTick
- [Cancel] button: subdued styling (transparent bg, border)
- The results list and divider are hidden while confirmPending (add `v-if="!confirmPending"` to divider and RecycleScroller)
- On cancel: restore previous search text and results list (confirmPending=false restores the v-show/v-if guards)
- ESC: existing `cancelConfirm()` call in onKeyDown already works — no logic changes needed
- Enter: existing `confirmAction()` call in onKeyDown already works (it fires on the document-level keydown, not on the button) — no logic changes needed
- The `showConfirm`, `cancelConfirm`, `confirmAction` functions are correct and unchanged
- `confirmBtnRef` ref is already declared — wire it to the new Confirm button

Purpose: The overlay approach had three unfixable bugs (position:fixed clipping, ESC handler scope, Cancel mousedown.prevent swallowing clicks). The inline approach inherits the working keyboard handler scope naturally.
Output: App.vue with inline confirmation row, no floating overlay.
</objective>

```typescript
const confirmBtnRef = ref<HTMLButtonElement | null>(null)
const confirmPending  = ref(false)
const pendingCommand  = ref<SearchResult | null>(null)

function showConfirm(item: SearchResult) {
  pendingCommand.value = item
  confirmPending.value = true
  nextTick(() => { confirmBtnRef.value?.focus() })
}
function cancelConfirm() {
  confirmPending.value = false
  pendingCommand.value = null
  inputRef.value?.focus()
}
async function confirmAction() {
  const item = pendingCommand.value
  if (!item) return
  confirmPending.value = false
  pendingCommand.value = null
  await hideWindow()
  await invoke('run_system_command', { cmd: item.id }).catch(console.error)
}
```
