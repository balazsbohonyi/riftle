**Test:** Open Settings, click the Global Shortcut field, press a key combination that is already taken by Windows or another application (e.g., `Ctrl+Space` if Windows IME is active, or try `PrintScreen`).

**Expected:** Same overlay but title reads "Restart Windows?" and the danger button reads "Restart".
**Why human:** Conditional copy rendering requires runtime verification.

**Expected:** Overlay closes, search text `> shutdown` is still present in the input, the shutdown item is still selected.
**Why human:** Query and selectedIndex preservation across overlay open/close cycle requires runtime observation.

### 5. Backdrop Click Does NOT Cancel

**Test:** With the confirmation overlay open, click outside the card (on the dark backdrop area).
**Expected:** Nothing happens — overlay stays open, backdrop click does not cancel.
**Why human:** `@mousedown.prevent` without handler behavior requires runtime validation.

### 6. Confirm Button Auto-Focus Allows Immediate Enter

## Summary

Phase 09.6 goal is fully achieved. Both safety improvements are substantively implemented and correctly wired:

**Atomic hotkey swap:** `update_hotkey` in `hotkey.rs` now uses register-new-first order. The old code unregistered the old hotkey before attempting the new one, creating a window where no hotkey was active. The new code calls `on_shortcut()` for the new key first — if it fails, `Err` is returned immediately and the old hotkey registration is untouched. The startup `register()` function with its Alt+Space fallback is unchanged.

**Settings.vue error feedback:** `hotkeyError` ref is wired into `onHotkeyChange` — cleared on each attempt, set with the Rust error message plus old hotkey name on failure, skipping `saveSettings()`. The `settings.value.hotkey = oldHotkey` assignment propagates through KeyCapture's prop watcher to revert the displayed key automatically.

**Confirmation overlay:** `CONFIRM_REQUIRED` Set gates only `system:shutdown` and `system:restart`. `launchItem` intercepts these before the `run_system_command` invoke. The Escape and Enter keyboard handlers are correctly prioritized. The backdrop `@mousedown.prevent` guards focus-loss auto-hide. The `launcher-show` listener resets overlay state on re-summon.

Six items require human verification (visual, focus, and runtime interaction behaviors).
