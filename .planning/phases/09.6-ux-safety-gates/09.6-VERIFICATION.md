## Root Cause of Remaining Gap

Plan 05 objective stated: "ESC and Enter keyboard handling already correct in logic — will work once focus stays within existing keyboard handler scope."

This assumption was incorrect. The `onKeyDown` handler is attached to the `<input>` element via `@keydown="onKeyDown"` (App.vue line 547). When `confirmPending=true`, the input is unmounted (`v-if="!confirmPending"`). The confirm button receives focus, but keydown events on the button bubble up through the DOM: confirm-btn → confirm-actions div → confirm-row div → search-area div → launcher-app div. None of these elements have a `@keydown` listener. The events reach the document root without any handler.

The fix requires one of:
1. Add `@keydown="onKeyDown"` to the `.launcher-app` root div in the template
2. Add `document.addEventListener('keydown', onKeyDown)` in `onMounted` and remove it in `onUnmounted`
3. Add explicit `@keydown.esc="cancelConfirm"` and `@keydown.enter="confirmAction"` to the confirm-row div

Option 1 is the minimal change — it makes `onKeyDown` catch all keydowns that bubble to the launcher root, which is already the effective target for all user interaction.

## Gaps Summary

**One root cause, three truth failures:**

The confirm-row keyboard scope gap (Escape does not work, Enter works only by browser default, Cancel accessible only via Tab+Enter or mouse) all share a single root cause: `onKeyDown` is bound to the `<input>` element only, and the input is unmounted when `confirmPending=true`.

The fix is a one-line template change: add `@keydown="onKeyDown"` to the `.launcher-app` root div (App.vue line 508), or alternatively move the listener to the document level. This would make the working `confirmPending` logic in `onKeyDown` (lines 272-275, 293-295) actually reachable during confirmation.

The three plans from gap closure (03-05) successfully closed 8 of the 11 truths. The remaining gap was not identified during plan execution because Plan 05 incorrectly assumed the keyboard handler scope would be sufficient after the redesign.
