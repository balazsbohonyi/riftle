# Phase 09.6 Plan 04: Settings HWND Subclass for SC_KEYMENU Suppression Summary

**SetWindowSubclass hook on the settings window HWND swallows WM_SYSCOMMAND SC_KEYMENU so Alt+Space can be captured as a hotkey without triggering the OS system menu**

- **Tasks:** 1

## Accomplishments
- Added `Win32_UI_Controls` to `windows` crate features in Cargo.toml
- Installed `SetWindowSubclass` on the settings window HWND in the Tauri `setup` callback (after the existing DWM block)
- `subclass_proc` swallows any `WM_SYSCOMMAND` where `wParam & 0xFFF0 == 0xF100` (SC_KEYMENU / Alt key activation) and returns `LRESULT(0)`, all other messages forwarded via `DefSubclassProc`
- `cargo build` succeeds; `cargo test` 76 passed, 0 failed, 2 ignored

## Decisions Made
- `SetWindowSubclass` and `DefSubclassProc` are in `windows::Win32::UI::Shell` (not `Win32::UI::Controls` as stated in the plan interfaces block). Since `Win32_UI_Shell` was already a feature, the API was already compiled in. The `Win32_UI_Controls` feature was still added to Cargo.toml as specified by the plan, but the import path in lib.rs was corrected to `Win32::UI::Shell`.
