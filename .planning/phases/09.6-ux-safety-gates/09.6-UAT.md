## Tests

### 1. Hotkey registration failure — inline error
expected: Open Settings → Hotkey section. Try changing the hotkey to a key already claimed by another app or OS (e.g. Win+D, PrintScreen, or any key Windows reserves). The KeyCapture input should revert to the previous hotkey automatically, and a red error message should appear below the Hotkey row saying something like "'<key>' could not be registered" and "still using <old hotkey>".
result: issue
reported: "the KeyCapture input reverts, but I don't see the error message"
severity: major

### 2. Successful hotkey change
expected: In Settings → Hotkey, change to an available hotkey (e.g. Alt+F9 or Ctrl+Alt+R). No error message shown. Close Settings. The new hotkey should summon/hide the launcher.
result: issue
reported: "Ctrl+Alt+Space worked. But when trying to reset back by pressing Alt+Space in the KeyCapture input, the OS window context menu appeared instead of capturing Alt+Space"
severity: major

### 3. Shutdown confirmation overlay
expected: In the launcher, type `> shutdown` and press Enter. Instead of executing immediately, a modal overlay should appear with a dark backdrop and a card asking "Shut down Windows?" The confirm (danger) button should be auto-focused so pressing Enter again would confirm.
result: issue
reported: "The overlay appears but is clipped at the top by the launcher field (should work like the context menu which is not clipped). ESC key did not dismiss it. Tab could switch to Cancel button but pressing Enter on Cancel did nothing."
severity: major

### 4. Restart confirmation overlay
expected: Type `> restart` and press Enter. A confirmation overlay should appear with "Restart Windows?" and a focused confirm button.
result: issue
reported: "Same problems as shutdown: overlay clipped at top, ESC didn't work, Cancel button unresponsive"
severity: major

### 5. Escape cancels confirmation overlay
expected: While the confirmation overlay is visible, press Escape. The overlay disappears, focus returns to the search input, and no system command is executed.
result: issue
reported: "ESC key did not dismiss the overlay (confirmed during test 3 and 4)"
severity: major

### 6. Cancel button dismisses overlay
expected: Open the shutdown confirmation overlay (> shutdown + Enter). Click the Cancel button. The overlay closes without executing anything.
result: issue
reported: "Cancel button focused via Tab but pressing Enter on it did nothing (confirmed during test 3)"
severity: major

### 7. Outside click does not dismiss overlay
expected: While the confirmation overlay is open, click on the dark backdrop area outside the card. The overlay stays open — it should NOT close or cancel on an outside click.
result: skipped
reason: Overlay design is being replaced entirely — not applicable to new design

### 8. Lock and Sleep bypass confirmation
expected: Type `> lock` and press Enter → executes immediately (screen locks) without any confirmation overlay. Same for `> sleep` — runs without a dialog.
result: skipped
reason: Cannot safely test during UAT; will verify after redesign is implemented
