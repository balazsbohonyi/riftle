## Gaps Summary

No gaps. All automated checks pass. Phase goal achieved:

1. The blocking `run_full_index` call has been removed from `lib.rs` — `grep "run_full_index" lib.rs` returns no matches.
2. The sync bug is fixed — every code path that triggers a background index (`try_start_index`, used by timer, watcher, and deferred startup thread) now calls `crate::search::rebuild_index(&app)` after `run_full_index` completes.
3. The deferred startup strategy (immediate on first run, 30s delay on subsequent) is implemented and wired correctly through `count_apps` and `app_count`.
4. The `TimerMsg::Reset` signal is sent after the deferred startup index so the periodic timer counts from actual index completion.
5. All unit tests compile and pass — the two tests that called `try_start_index` directly were rewritten to test the underlying `AtomicBool` CAS semantics without needing a `tauri::AppHandle`.
