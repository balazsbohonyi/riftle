# Phase 09.8 Plan 02: Improve Indexing Performance — Non-Blocking Startup Summary

**Non-blocking startup via deferred index: window appears instantly, background index fires immediately on first run or after 30s on subsequent runs, and every background index now refreshes the in-memory SearchIndexState**

## Accomplishments
- Removed the blocking `run_full_index()` call from lib.rs startup — window is now interactive immediately
- Fixed the sync bug where background re-indexes updated SQLite but never refreshed `SearchIndexState` — `try_start_index` now calls `rebuild_index` after each run
- Added deferred startup thread in `start_background_tasks`: fires immediately when DB is empty (first run), or after 30-second delay when DB has data
- Sent `TimerMsg::Reset` after deferred startup index so the auto-index interval counts from indexing completion, not app launch
- Rewrote two broken unit tests to test `AtomicBool` CAS guard semantics directly (no `AppHandle` needed in unit tests)
