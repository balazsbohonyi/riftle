## Tests

### 1. Settings Shortcut Editing
expected: In Settings, the Shortcuts section has Directories and Files tabs. You can add a folder shortcut and a file shortcut, edit their alias/path/parameters where applicable, save them, close Settings, reopen Settings, and the saved shortcuts are still present with the same values.
result: pass

### 2. Shortcut Validation
expected: Settings refuses to save invalid shortcuts. Duplicate display names are rejected, file shortcut parameters require an alias when needed, parameters are rejected for non-executable document shortcuts, and invalid paths are blocked while editing existing shortcuts.
result: pass

### 3. Shortcut Search Results
expected: Searching by a shortcut alias or fallback name shows matching directory and file shortcuts above normal app results. Shortcut search is prefix-based, case-insensitive, and system command searches starting with ">" still show only system commands.
result: pass

### 4. Shortcut Launch Behavior
expected: Pressing Enter on a directory shortcut opens the folder and hides the launcher. Pressing Enter on a file shortcut opens through the intended app/default association path and hides the launcher only after a successful launch.
result: pass

### 5. Shortcut Failure Warnings
expected: Launching a shortcut whose target no longer exists or cannot be opened shows a warning in the launcher and keeps the launcher visible so the user can correct the problem or continue searching.
result: pass

### 6. Shortcut Display And Icons
expected: Shortcut entries in the launcher use useful native-looking icons instead of the generic Tauri icon. Directory shortcuts use a folder icon, while file shortcuts use the associated app icon or the explicit executable app icon when one is configured.
result: pass

### 7. Shortcut Settings Layout
expected: The Shortcuts settings UI is compact. Read-only entries show alias/fallback, path, and file parameters only when present; edit/remove actions appear on hover; Directories and Files tabs split the width evenly; lists scroll cleanly without layout width jumps.
result: pass

### 8. Search And Shortcuts Explanations
expected: The Search and Shortcuts sections include explanatory text below the heading divider. Search clearly says additional paths are extra locations beyond Riftle's default Windows app scan locations, while Shortcuts explains that named folders/files/app launches appear directly in search results.
result: pass
