## Decisions Made

- Shortcut IDs use `shortcut:dir:{hash}` and `shortcut:file:{hash}` prefixes with deterministic path hashing so later search/launch work can share one contract.
- Parameterized shortcuts are restricted to `.exe`, `.com`, `.bat`, and `.cmd`; `.lnk` is deliberately excluded until a later phase defines argument handling.
- Parameterized executable shortcuts require an alias, preventing duplicate or unclear fallback names for command variants.
