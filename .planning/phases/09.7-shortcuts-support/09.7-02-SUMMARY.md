## Decisions Made

- Shortcut matching is prefix-only and case-insensitive, using `shortcut_display_name()` so aliases win and empty aliases fall back to basename/filename stem.
- Shortcut rows use `kind: "shortcut_dir"` and `kind: "shortcut_file"` with `icon_path: "generic.png"` to stay compatible with existing icon validation.
- Settings load failures are non-fatal for search; fatal settings recovery errors log and return app-only results.
