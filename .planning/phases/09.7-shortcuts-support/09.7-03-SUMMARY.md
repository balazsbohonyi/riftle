## Decisions Made

- Shortcut launch failures return `Ok(ShortcutLaunchResult { success: false, warning: Some(...) })` instead of `Err`, so expected OS launch failures can be rendered by the frontend without being treated as IPC transport failures.
- Rust does not hide the launcher for shortcut launches. Plan 04 owns frontend behavior and can hide only when `success` is true.
- The Open With dialog uses the existing `windows` crate Shell bindings already enabled in `Cargo.toml`; no new dependency or feature expansion was needed.
