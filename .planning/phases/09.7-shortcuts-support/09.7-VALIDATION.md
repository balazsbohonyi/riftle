## Manual-Only Verifications

| Behavior | Requirement | Why Manual | Test Instructions |
|----------|-------------|------------|-------------------|
| Native folder picker adds a directory shortcut | Settings editing | Tauri dialog plugin requires runtime UI | Open Settings, add a directory shortcut, save, reopen Settings, confirm it persists. |
| Native file picker adds file shortcuts | Settings editing | Tauri dialog plugin requires runtime UI | Add a document and executable shortcut, save, reopen Settings, confirm fields persist. |
| Directory shortcut launch hides launcher on success | Launch behavior | OS shell integration and window hide behavior | Search shortcut alias, press Enter, verify folder opens and launcher hides. |
| Non-executable file launch uses default app or Open With | Launch behavior | Depends on local file associations | Add a file with and without an association, verify default open or Open With flow. |
| Parameterized executable launch passes raw arguments | Launch behavior | Requires local executable target | Add an executable shortcut with a quoted path argument, launch, verify the app receives the argument. |
| Missing target warning keeps launcher open | Failure behavior | User-visible window state | Add a shortcut to a removed path, launch it, verify warning appears and launcher stays visible. |
