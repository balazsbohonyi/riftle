Create `src/Settings.vue` — the full settings window root component. Requirements per section:

**Window structure:**
- Root element: `<div class="settings-app">` with `data-theme` attribute mirroring current settings.theme (so the settings window itself also themes correctly)
- Custom header: `<div class="settings-header" data-tauri-drag-region>` — contains left side (magnifier/icon + "Riftle Settings" title) and right side (close `×` button calling `getCurrentWindow().close()`)
- Scrollable content area: `<div class="settings-content">` wrapping all four `<Section>` components

**onMounted:**
- Set `isTauriContext` check
- Call `invoke('get_settings_cmd')` to load all settings into a reactive `settings` ref
- If installed mode, call `isEnabled()` from plugin-autostart to sync autostart state
- Apply initial theme to document root (for settings window itself): `document.documentElement.setAttribute('data-theme', settings.theme)` if not 'system'

**General section (SETT-04):**
- `<Row label="Launch at startup" :hint="isPortable ? 'Not available in portable mode' : undefined">`
- `<Toggle v-model="settings.autostart" :disabled="isPortable" @update:modelValue="onAutostartChange" />`
- `onAutostartChange(v)`: if portable return; if v call `enable()` else `disable()`; then call `saveSettings()`

**Hotkey section (SETT-05):**
- `<Row label="Global shortcut">`
- `<KeyCapture v-model="settings.hotkey" @change="onHotkeyChange" />`
- `onHotkeyChange(hotkey)`: call `invoke('update_hotkey', { hotkey })`; update `settings.hotkey`; call `saveSettings()`
- Note: update_hotkey already persists to settings.json internally, but call saveSettings() too for consistency

**Search section (SETT-06):**
- `<Row label="Additional paths"><PathList v-model="settings.additional_paths" @change="onPathsChange('additional_paths', $event)" /></Row>`
- `<Row label="Excluded paths"><PathList v-model="settings.excluded_paths" @change="onPathsChange('excluded_paths', $event)" /></Row>`
- `<Row label="Re-index interval">` with a `<select>` dropdown: options 5/15/30/60/0 (displayed as "5 min"/"15 min"/"30 min"/"60 min"/"Manual only"); `@change="onIntervalChange"`
- `<Row label="Re-index now">` with `<button @click="onReindexNow">Re-index</button>`
- `onPathsChange(field, paths)`: update settings[field], call saveSettings(), call invoke('reindex') (path changes require fresh index)
- `onIntervalChange(e)`: update settings.reindex_interval, call saveSettings(), emit settings-changed with reindex_interval
- `onReindexNow()`: call `invoke('reindex')`; show brief visual feedback (button text changes to "Indexing…" then back after 1s)

**Appearance section (SETT-07):**
- `<Row label="Theme">` with `<select v-model="settings.theme" @change="onThemeChange">` options: system/light/dark
- `<Row label="Opacity">` with `<input type="range" :min="0.85" :max="1" :step="0.01" v-model.number="settings.opacity" @input="onOpacityInput" />`; display numeric value beside slider
- `<Row label="Show path">` `<Toggle v-model="settings.show_path" @update:modelValue="onShowPathChange" />`
- `onThemeChange()`: apply theme to settings window root (`document.documentElement`); call saveSettings(); emitTo('launcher', 'settings-changed', { theme: settings.theme })
- `onOpacityInput()`: call saveSettings() (debounced or direct); emitTo('launcher', 'settings-changed', { opacity: settings.opacity }) — live while dragging
- `onShowPathChange(v)`: update settings.show_path; call saveSettings(); emitTo('launcher', 'settings-changed', { show_path: v })

**saveSettings() helper:**
```typescript
async function saveSettings() {
  if (!isTauriContext.value) return
  const { system_tool_allowlist, animation, ...rest } = settings.value  // preserve existing fields
  await invoke('set_settings_cmd', { settings: { ...settings.value } }).catch(console.error)
}
```
