Two changes to src/App.vue to make the window resize animation visible:

**Change 1 — CSS height transition on .result-list** (in the `<style>` block, around line 418-423):

Add `transition: height 180ms ease` to `.result-list`. This lets the browser animate the DOM height change driven by the `:style="{ height: listHeight + 'px' }"` binding BEFORE the OS window catches up:

```css
.result-list {
  overflow-y: auto;
  overflow-x: hidden;
  scrollbar-width: none; /* Firefox */
  transition: height 180ms ease;
}
```

**Change 2 — Defer setSize() after CSS animation** (in `updateWindowHeight`, around lines 69-78):

Delay the `setSize()` call so the CSS animation plays first, then the OS window resizes to match. The delay should respect the current animMode setting:
- slide: wait 180ms
- fade: wait 120ms
- instant: no delay (0ms)

```typescript
async function updateWindowHeight() {
  if (!isTauriContext.value) {
    console.log('[App] updateWindowHeight skipped: not in Tauri context')
    return
  }
  const h = Math.max(56 + listHeight.value, 56)
  console.log('[App] updateWindowHeight:', { listHeight: listHeight.value, totalHeight: h })
  // Delay OS window resize until after the CSS height transition completes
  const delay = animMode.value === 'slide' ? 180 : animMode.value === 'fade' ? 120 : 0
  if (delay > 0) {
    await new Promise(resolve => setTimeout(resolve, delay))
  }
  await getCurrentWindow().setSize(new LogicalSize(500, h)).catch(console.error)
}
```

These two changes work together: the CSS transition plays on the DOM element immediately when listHeight changes, then after the animation completes the Tauri window is resized to the new logical height.

Fix the virtual scroll selection highlight inconsistency by destructuring `active` from the RecycleScroller v-slot scope and using it to guard the `:class` binding and `@mousemove` handler.

In src/App.vue, locate the RecycleScroller template (around lines 270-285). Make three targeted changes:

**Change 1 — Destructure active from v-slot:**

Current:
```html
v-slot="{ item, index }"
```

Replace with:
```html
v-slot="{ item, index, active }"
```

**Change 2 — Gate :class on active:**

Current:
```html
:class="{ selected: index === selectedIndex }"
```

Replace with:
```html
:class="{ selected: active && index === selectedIndex }"
```

This prevents the `selected` class from being applied to DOM nodes that are mid-recycle (active=false), which caused the blinking selection highlight during scroll.

**Change 3 — Guard @mousemove on active:**

Current:
```html
@mousemove="selectedIndex = index"
```

Replace with:
```html
@mousemove="active && (selectedIndex = index)"
```

This prevents stale slot indices from corrupting selectedIndex when the mouse moves during scroll-triggered DOM recycling.

These are the only changes in this task — do not touch any other part of the template or script.
